# Time Complexity Analysis

### Bubble Sort

Bubble Sort repeatedly compares adjacent elements and swaps them if they are in the wrong order.

Time Complexity:

- Best Case: O(n)
- Average Case: O(n²)
- Worst Case: O(n²)

### Quick Sort

Quick Sort uses the divide-and-conquer approach and partitions the array around a pivot element.

Time Complexity:

- Best Case: O(n log n)
- Average Case: O(n log n)
- Worst Case: O(n²)

## Comparison

Bubble Sort is simple but becomes slow when the number of elements increases. Quick Sort is more efficient and performs much better for large datasets, which is why it is widely used in real-world applications.
